Tutorial and spec's

I took the all the freedom that was given to me with interpreting the requirements, hence here are some specifications in order for a better user experience to be achieved

The task list

This is a view of all pending tasks. By clicking process you are sending them to a process queue. After the processing is finished, a push notification is sent to all subscribed clients so they can remove the processed task and prevent a stale view. You can try this with multiple tabs/browser windows.

The scheduler

While you are enjoying this beautifull CSS marvel, a scheduler is creating tasks in the background. New tasks created by the scheduler will be pushed on top of this list view. If you don't want to wait for that to happen, you can force a random task creation via this link. It works even when called from another tab

The details button

There is far more in a task then it meets the eye on this screen. Click the details button to go to a more detailed view of a task that includes editing options as well as all the fields from the requirements (Status,Priority,CreatedAt,UpdatedAt,UUID,DueDate....etc)

The process button

Clicking on process will send the task to a queue that will eventually process it. Once it has been processed it will be removed from all active clients via a push notification. Feel free to try this from diferent browsers/tabs.

The postpone button

Clicking postpone will send you to a screen for choosing a postpone date. Back in the first paragraph I gently mislead you. The list of awesomness doesn't show all the tasks. It ONLY shows pending tasks which have not been postponed to a date after now. A scheduler takes care of all clients getting a push notification when such postponed task becomes due.
